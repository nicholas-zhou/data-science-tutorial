Introduction¶

This is a tutorial that walks through the steps of the data lifecycle: data collection, data processing, exploratory analysis & data visualization, hypothesis testing & machine learning, and insight & policy decision. In data science, it is very important to understand data, process it, extract value from it, visualize it, and communicate it. Hopefully by the end of the tutorial, you walk away with a full understanding of the lifecycle and how to apply it!

Context¶

There are people who try to argue that the United States has more immigrants originating from poorer countries rather than wealthier countries. We wanted to test whether or not this is true using the different steps of the data lifecycle. Looking at the GDP per capita between 2013 and 2015 for different countries, as well as their number of immigrants (to the United States), we can test the validity of this argument.

There is also another argument that the US should accept immigrants from more developed countries since their citizens are seen as more "desirable" compared to those from underdeveloped countries. With this argument comes the question "why would someone decide to immigrate from a developed country to the United States?". While there are many reasons for someone to immigrate, one of the most common reasons, especially in the United States, is for economic opprortunity. We decided to see the correlation between the national origin of immigrants to the United States and the economic opprotunity of those immigrants' home country (which we decided to measure by GDP per capita).

Data Collection¶

Of course, we want to start by loading in the data. There is a lot of data on the internet available for us to use.

The data set for GDP per capita that we used can be found from Our World in Data. The data can be found under the "Catch up growth can be fast" section. If you click DOWNLOAD on the chart to the right of the section, you can download the file as a CSV. In addition to the data set, this website has a lot of great information on the history of economic growth if you are interested in reading more about it.

We also decided on using data from Data.world for number of immigrants. The website requires an account to access the data, but once you create one, you will be able to download the file dhs.xlsx. Notice the file has an Excel sheet extension. To convert that to a CSV (Comma-Separated Values) file, open it and go to Table3 at the bottom (it should be the page with Region of birth in cell A1). Then navigate to File -> Export -> Change File Type -> CSV. A popup will tell you that you cannot export multiple sheets. You can click OK and it will only save Table3, which is all we need.

Two other sources we considered were The Department of Homeland Security and Migration Policy Institute. While the DHS would have been a great source to get immigration numbers, we decided against it because the data for different years were separated into different files. We also could not use the Migration Policy Institute's data because the Excel sheet was formatted, so we could not convert to a CSV file.

Now that we have the data files, we can bring them into our notebook. Pandas has a function that allows us to read straight from a CSV file and put that data into a DataFrame. DataFrames are two-dimensional data structures similar to SQL tables or spreadsheet data representations. The two CSV files should be in the same folder as the notebook.

However, the names of the countries above are the more standard ones today (i.e. Myanmar is former Burma), we want to change the names in the immigration DataFrame to match the ones in the GDP per capita DataFrame.

In order to do this, we can usea dictionary to map the old names (from the immigration DataFrame) to new names (from the GDP per capita DataFrame). Then, we will go through every 'Region of birth' entry in the immigration DataFrame, and update it if it has the same name as one of the key values in the dictionary.

Merging DataFrames¶

Now we can finally merge the GDP per capita DataFrame into the immigration DataFrame. To do this, we create three new columns in our immigration DataFrame (one for each year). The columns will consist of the GDP per capita for that row's country according to the GDP per capita DataFrame.

Missing Data¶

We noticed that for the 2013 column, Comoros had a number of immigrants of "D". Based on the low immigration numbers from Comoros in 2014 and 2015, we inferred that "D" was a substitute value for none. Since we want to be able to convert the values to integers, we will have to address this one value. There are a couple options: remove the row or encode the missing data. We decided it would be best to encode the data and change it to 0. This is because Comoros has data for 2014 and 2015, so we do not want to remove it entirely. Additionally, its immigration numbers for 2014 and 2015 are pretty low, so it could factor into how few people immigration from this country.

In [75]:
# encode to 0
immigration_data.at[immigration_data[immigration_data['Region of birth'] == 'Comoros'].index[0], '2013'] = '0'

# for every entry in the 2013, 2014, and 2015 columns
for year in ['2013', '2014', '2015']:
    arr = []
    for i in immigration_data[year]:
        # remove commas and convert to integer
        arr.append(int(i.replace(',', '')))
    immigration_data[year] = arr

Exploratory Data Analysis and Data Visualization¶

Exploratory Data Analysis is the process of finding patterns and the main characteristics of a dataset in order to begin making hypotheses about the data. It is very useful to do this along with Data Visualization because plots and graphs allow you to visualize the patterns. We have a few different options for how we want to visualize the data. The first one to consider is making line graphs for the GDP per capita for each country. However, this would not really show us anything significant, because we cannot see how the GDP per capita affects the number of immigrants.

Scatter Plots¶

Another option is to use scatter plots of GDP per capita vs number of immigrants for every year. This is much better, since we have both pieces of data that we are looking at in the plots. We can also plot a regression line to see if we can find any patterns in the three scatter plots. Pandas has plotting functions and the regression lines will be added using NumPy.

An alternative to using Pandas and NumPy to make the scatter plots would be using Seaborn. Plotting a scatter plot on Seaborn automatically plots the regression line for you. We chose Pandas and NumPy because we had more experience with it and we knew how to change the colors on the plots.

Initial Thoughts from Plots¶

In each of the three plots, we can see that that for the regression line, the number of immigrants decreases as the GDP of countries increases. Based off of this, we think there could be a correlation between GDP per capita of a country and the number of immigrants from that country to the US, but we cannot be certain until we perform hypothesis testing.

Something else to note is there are two clear outliers in Mexico and Qatar. Mexico looks to be a country with low GDP per capita and a lot of immigrants coming to the US, while Qatar has a high GDP per capita with not many immigrants coming to the US.

Hypothesis Testing¶

Hypothesis testing is where tests are used to compare a hypothesis to a null hypothesis. A null hypothesis is that there is no significant correlation and it is rejected if its probability is below a standard level (0.05). For our situation, the null hypothesis would be there is no correlation between a country's GDP per capita and the number of immigrants the U.S. gets from it, i.e. people immigrate to the U.S. despiite the economic status of their country of origin. The alternative hypothesis (alternative to the null hypothesis) would be there is a correlation between the GDP per capita of a country and the amount of U.S. immigrants from there, i.e. citizens from less economically developed countries are more likely to immigrate to the U.S.

To test this we will run linear regression line tests on our data.

Insights and Final Thoughts¶

Recap¶

From this tutorial, we wanted to learn about whether there is a correlation between the wealth of certain countries and the number of immigrants the United States receive from those countries. We began by collecting data from a couple different sources and organizing the data in a structure that would be easy to manipute. Then, we plotted the data and performed some hypothesis testing to see whether or not there is a correlation between GDP per capita and number of immigrants to the US. Let's try to put this all together now.

Conclusion¶

When first started this project, we wanted to see if economic opportunity was a major influence for immigration to the United States. We decided to do this by collecting data on the national origins of immigrants to the United States and the GDP per capita and running a linear regression test to see if there was a significant correlation between the two. We also decided to use several years so that way we would have multiple samples and don't base our conclusion on just one year of data.

When we plotted the data and fit a linear regression line, we saw that there was some negative correlation between the two fields. However, when we ran the significance test we saw that there was no significant correlation between GDP per capita and the amount of immigrants the U.S. gets from each country.

Things to Consider¶

If you can remember, we removed some countries from the data sets at the beginning. We did this because the two sets did not have the same countries, and it was really difficult finding sets with similar ways of organizing their data. Something to consider is there is a chance those countries could reveal something we did not account for, or at the very least, just provide more data to work with. We could have also included a bigger range of years, which could provide more context to what was happening around the world in terms of immigration during certain periods.
